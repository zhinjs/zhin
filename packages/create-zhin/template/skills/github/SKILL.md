# GitHub

Use this skill when preparing issue reports, PR descriptions, release notes, or review responses.

## Workflow

1. Capture the exact problem, environment, reproduction steps, and expected behavior.
2. For PRs, include summary, key changes, validation, and risk.
3. Keep credentials, tokens, and private data out of the text.
4. Link local evidence or command output only when it is relevant.

## Output

Produce ready-to-paste GitHub text. If publishing is requested, use the available GitHub tooling or `gh` inside the project.
